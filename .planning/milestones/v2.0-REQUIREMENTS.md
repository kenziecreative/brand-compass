# Requirements: v2.0 Multi-Bundle Output

**Source:** PRD from Claude Webb, reviewed and adjusted by lead strategist and developer.

## Context

Brand Compass produces a single output bundle today. Two new consumers exist:
1. Claude Code/Cowork load skills — a well-formed skill lets agents write on-brand
2. Claude Design ingests design system codebases during org onboarding

This milestone adds two new output bundles generated from existing brand data. No changes to discovery phases 0-7.

## PRD Adjustments (agreed with developer)

1. **Client specimens stay self-contained** — inline tokens preserved for client/ bundle. Design kit specimens link to external tokens. Different consumers, different constraints.
2. **ui-kit.html stays standalone** — no assembly relationship between client ui-kit.html and atomized design-kit components.
3. **D6 (PPTX generation) deferred** — introduces Node dependency, build step, marginal value since Claude Design would create decks from the design kit anyway.
4. **Phase 8 skill file added to integration scope** — it orchestrates assembly and needs to know about new agents.
5. **Agent model/tools specified** — both new agents are sonnet with Read, Write, Glob, Grep.
6. **Untestable acceptance criteria replaced** — brand-prompt.md voice-match test replaced with structural checks.
7. **Frontend path changes accounted for** — React dashboard has hardcoded output paths that must update.
8. **Voice-fingerprint backlog item fixed** — Document Assembler gets voice-fingerprint.md input while we're modifying it.

## Out of Scope

- Changes to discovery phases 0-7
- Framework-specific component libraries (React, Vue, Svelte)
- Light/dark token pairs unless Phase 7 specifies dark mode
- Figma plugin beyond passive DTCG interop
- PPTX generation (deferred — Claude Design creates decks from the design kit)
- Automated publishing to live URL
- Per-project brand prompt beyond what ships in skill bundle
